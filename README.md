# Exam-LLM: textual proximity between student answers and LLM-generated answers

This repository contains the code and input-data structure used to analyse textual proximity between written student exam answers and answers generated by large language models (LLMs).

The method is intentionally transparent: it uses TF-IDF representations with unigrams and bigrams, cosine similarity in the original TF-IDF space, and t-SNE only as a two-dimensional visualization layer. The approach is designed as an exploratory and pedagogical tool, not as a stand-alone detector of misconduct.

## Input data

The analysis expects LLM files directly in `data/` and student files in `data/students/`.

For example:

```text
data/chatgpt10.txt
data/claude10.txt
data/deepseek10.txt
data/gemini10.txt
data/mistral10.txt
data/grok10.txt
data/students/E01.txt
data/students/E02.txt
```

Each text file should contain answer blocks labelled as:

```text
R1
...
R2
...
R3
...
R4
...
```

LLM files may contain several variants labelled as `COPIE A`, `COPIE B`, etc. These variants are automatically split by the script.

## Ethical and data-use note

The repository may include LLM-generated answers and anonymized student answers for transparency and reproducibility. Student answers must be anonymized before release. These data must not be used to identify, profile, rank, or sanction individual students.

The method should be interpreted as a similarity-based exploratory visualization. It is not a proof of AI use and should not be used as the sole basis for academic misconduct decisions.

Do not commit the anonymization correspondence file:

```text
student_anonymization_map.csv
```

This file is ignored by `.gitignore`.

## Installation

Create a Python environment and install the required packages:

```bash
pip install -r requirements.txt
```

Alternatively, with conda:

```bash
conda env create -f environment.yml
conda activate exam-llm
```

## Running the analysis

For the corpus with 10 LLM answers per model:

```bash
python src/llm_exam_proximity.py \
  --data-dir data \
  --students-dir data/students \
  --n-llm-responses 10 \
  --output-dir outputs/output_10LLM
```

For 20 or 30 LLM answers per model:

```bash
python src/llm_exam_proximity.py --n-llm-responses 20 --output-dir outputs/output_20LLM
python src/llm_exam_proximity.py --n-llm-responses 30 --output-dir outputs/output_30LLM
```

Or use the helper scripts:

```bash
bash scripts/run_10_responses.sh
bash scripts/run_20_responses.sh
bash scripts/run_30_responses.sh
```

## Main outputs

The script writes the following files to the selected output directory:

```text
Figure_all_questions_4panels.png
Figure_all_questions_4panels.pdf
scores_R1.csv
scores_R2.csv
scores_R3.csv
scores_R4.csv
scores_all_questions.csv
summary_all_questions.csv
R1_full_similarity_matrix.csv
R1_tsne_coordinates.csv
...
```

The main metrics include:

- `best_similarity_to_ai`: highest cosine similarity between a student answer and an LLM answer;
- `closest_ai_model`: LLM model associated with the closest answer;
- `ai_in_topk`: number of LLM answers among the top-k nearest neighbours;
- `mean_similarity_to_ai`: mean similarity between the student answer and all LLM answers;
- `mean_similarity_to_students`: mean similarity between the student answer and other student answers;
- `ai_to_students_similarity_ratio`: ratio between mean LLM similarity and mean student similarity.

## Method summary

For each question (`R1` to `R4`), the workflow is:

1. read LLM and student text files;
2. extract response blocks using the `R1`--`R4` labels;
3. anonymize student identifiers for plotting and scoring;
4. compute TF-IDF vectors using unigrams and bigrams;
5. compute cosine similarities in TF-IDF space;
6. rank student answers by proximity to LLM-generated answers;
7. compute a t-SNE projection for visualization;
8. export figures, similarity matrices, and score tables.

## Reproducibility notes

The t-SNE random seed is fixed in the script (`TSNE_RANDOM_STATE = 42`). Similarity scores are computed before t-SNE, directly in TF-IDF space, and are therefore not affected by the two-dimensional projection.
