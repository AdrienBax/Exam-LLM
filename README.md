# AI-Exam LLM Proximity Analysis

Transparent scripts and input-data structure for comparing written exam answers from students with reference answers generated by several large language models (LLMs).

The repository supports the analysis used in the manuscript on AI use in written remote exams. The central idea is not to build a black-box AI detector, but to provide a reproducible, interpretable workflow that helps visualize the lexical and structural proximity between student answers and LLM-generated answers.

## What the code does

For each exam question (`R1` to `R4`), the main script:

1. reads student answers and LLM reference answers from plain-text files;
2. extracts the answer blocks marked as `R1`, `R2`, `R3`, and `R4`;
3. anonymizes student identifiers as `E01`, `E02`, etc.;
4. preserves selected seeded/test labels such as `META`, `ALIBABA`, `IBM`, `KIMI`, and `CHEAT`;
5. represents texts with TF-IDF using unigrams and bigrams;
6. computes cosine similarity in the original TF-IDF space;
7. ranks students according to their closest LLM answer;
8. projects the corpus with t-SNE for visualization;
9. exports figures, score tables, similarity matrices, and t-SNE coordinates.

## Installation

Using `pip`:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Using `conda`:

```bash
conda env create -f environment.yml
conda activate ai-exam-llm-proximity
```

## Quick start

### Run the analysis with 10 LLM answers per model

```bash
python src/llm_exam_proximity.py \
  --data-dir data \
  --students-dir data/students \
  --n-llm-responses 10 \
  --output-dir outputs/output_10LLM
```

or:

```bash
bash scripts/run_10_responses.sh
```

### Run with 20 or 30 LLM answers per model

```bash
bash scripts/run_20_responses.sh
bash scripts/run_30_responses.sh
```

## Main options

```bash
python src/llm_exam_proximity.py --help
```

Important arguments:

| Option | Description |
|---|---|
| `--data-dir` | Directory containing the LLM text files. |
| `--students-dir` | Directory containing student text files. |
| `--n-llm-responses` | Selects the LLM corpus size: `10`, `20`, or `30`. |
| `--output-dir` | Directory where figures and CSV files are written. |
| `--top-k` | Number of nearest neighbors used for the `ai_in_topk` metric. |
| `--show` | Display the generated figure interactively. |

## Outputs

For each question, the script writes:

```text
R1_tsne_coordinates.csv
R1_full_similarity_matrix.csv
scores_R1.csv
```

and similarly for `R2`, `R3`, and `R4`.

It also writes:

```text
Figure_all_questions_4panels.png
Figure_all_questions_4panels.pdf
scores_all_questions.csv
summary_all_questions.csv
student_anonymization_map.csv
```

The most important output variables are:

| Variable | Meaning |
|---|---|
| `best_similarity_to_ai` | Highest cosine similarity between a student answer and any LLM answer. |
| `closest_ai_model` | LLM model associated with the closest reference answer. |
| `ai_in_topk` | Number of LLM answers among the top-k nearest neighbors of a student answer. |
| `mean_similarity_to_ai` | Mean similarity between a student answer and all LLM answers. |
| `mean_similarity_to_students` | Mean similarity between a student answer and other student answers. |
| `ai_to_students_similarity_ratio` | Ratio between mean LLM similarity and mean student similarity. |

## Methodological note

The similarity scores are computed in the original TF-IDF space using cosine similarity. The t-SNE projection is used only for visualization. Therefore, ranking tables and similarity matrices should be interpreted as the quantitative outputs, while the two-dimensional t-SNE figure should be interpreted as an exploratory map of proximity patterns.

The TF-IDF representation uses:

```python
TfidfVectorizer(
    lowercase=True,
    ngram_range=(1, 2),
    max_df=0.95,
    min_df=1,
    sublinear_tf=True,
)
```

This means that the current implementation uses unigrams and bigrams, but not trigrams or higher-order n-grams.

## Ethical and pedagogical use

This workflow should not be interpreted as definitive evidence that a student used an LLM. It is designed to support reproducible analysis, visualization, and discussion of answer proximity. Any use in an educational context should be complemented by human assessment, institutional rules, and appropriate safeguards for students.
